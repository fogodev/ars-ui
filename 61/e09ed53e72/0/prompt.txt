### LoadChildren effect / pending_load design as built
`Effect::LoadChildren` is a unit `Copy` effect (the `EffectName: Copy` constraint forbids `LoadChildren(Key)`). The key travels in the transient `Context.pending_load`, set alongside `load_state[key]=Loading` when an expanding branch is `NotLoaded`. Implemented inside `expanded_change_plan`, which now takes an explicit `expanding: &amp;[Key]` list (the keys the event expands) — fires for the first `NotLoaded` key, works for `ExpandNode`/`ToggleNode`/`ExpandAll`, fires for both controlled and uncontrolled, and retries on re-expand of a still-`NotLoaded` branch.

### ChildrenLoaded payload choice
`Vec&lt;TreeItemConfig&lt;TreeItem&gt;&gt;`. `TreeItem` does **not** implement `CollectionItem` (it has no key field — keys live on `Node`/`TreeItemConfig`), so the real `insert_child` (which requires `T: CollectionItem`) is unusable, and there is no public `Node&lt;T&gt;` constructor. `TreeItemConfig` is the public construction type for `TreeCollection` and carries key/text/value/children/default_expanded. I splice the loaded configs under `parent` and rebuild via `TreeCollection::new` using only the `T: Clone` public API (`all_nodes`/`children_of`/`is_expanded`) in a new private `insert_loaded_children` helper. This required adding `#[derive(Clone, PartialEq)]` to `TreeItemConfig` in `crates/ars-collections/src/tree_collection.rs:20` (conditional on `T`; the manual `Debug` is preserved) so it can ride in the `Clone + PartialEq` `Event` enum.
