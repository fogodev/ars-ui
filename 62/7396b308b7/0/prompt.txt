`Part::Chunk` is intentionally not a Part variant: the chunk anatomy slot is
parametric on a runtime boolean (`highlighted`), and `ConnectApi::part_attrs`
takes a `Part` by value with no payload. Adapters call `chunk_attrs(bool)`

`root_attrs()` emits `data-ars-scope="highlight"` + `data-ars-part="root"`
(via `Part::Root.data_attrs()`). `chunk_attrs(b)` emits
`data-ars-part="highlight-chunk"` + `data-ars-highlighted="true"|"false"`.

## Algorithm â€” `highlight_chunks`

Reuse `ars_i18n::to_lowercase(text, locale)` for Unicode-aware,
locale-sensitive normalisation. This is the **same** helper the
already-implemented `ars-collections::typeahead` (`typeahead.rs:255,297`) uses
for case-insensitive matching, so Turkic locales (`Ä°`/`i`/`I`/`Ä±`), German
eszett, Greek final sigma, etc. are already handled by the underlying ICU4X
`CaseMapper`. See "Open question â€” case folding" below.

Pipeline:

1. **Short-circuit empty input.** If `props.query.is_empty()` or every entry
   is empty, return `vec![HighlightChunk { text: &props.text, highlighted: false }]`.
   If `props.text.is_empty()`, return an empty `Vec`.
2. **Build normalised text + byte-mapping** (only when `ignore_case`):
   - Iterate `props.text.char_indices()`. For each char, take its substring
     (`text[orig_start..orig_end]`), pass to `ars_i18n::to_lowercase`, and
     append the lowercased bytes to a `String` while pushing `orig_start`
     into a parallel `Vec<usize>` once per appended byte. Push a sentinel
     `text.len()` at the end of the map.
   - The resulting `byte_map[i]` â‰ˇ original-text byte index of the source
     char that produced normalised byte `i`. Property:
     `byte_map[match_end_in_normalised]` is always an original-char boundary
     (or `text.len()` at the end), because each fold block is emitted
     contiguously.
   - When `!ignore_case`, the normalised text **is** the original text and
     `byte_map` is identity (`0..=text.len()`).
3. **Collect raw match ranges per query Ă— strategy:**
   - `Contains`: `normalised_text.match_indices(&normalised_query)`. Map each
     `(start, _)` to original range `(byte_map[start], byte_map[start + normalised_query.len()])`.
   - `StartsWith`: if `normalised_text.starts_with(&normalised_query)`, emit
     a single range `(0, byte_map[normalised_query.len()])`.
   - `Fuzzy`: iterate normalised text char-by-char, advancing a query-char
     cursor; each matched character emits one original-char range
     `(orig_start, orig_end)`. Use `char_indices()` on the **original** text
     and `to_lowercase` per-char to avoid the byte-map indirection for the
     1-char case (cleaner + handles Unicode expansion correctly).
4. **Merge overlapping / adjacent ranges.** Sort by start; sweep merging
   `(a, b), (c, d)` where `c <= b` into `(a, max(b, d))`. Adjacency-merge
   (`c == b`) keeps the rendered output minimal â€” naturally consolidates
   fuzzy 1-char hits that happen to land next to each other and meets spec
   Â§3.1's "consolidate adjacent highlighted chunks" recommendation.
5. **Build chunks.** Walk the merged ranges; emit a non-highlighted
   `HighlightChunk` for each gap and a highlighted one for each range. Skip
   zero-length segments so we never emit an empty chunk.

### Edge cases the algorithm must hit

- Empty query / all-empty queries â†’ one non-highlighted chunk (`text`).
- Empty text â†’ empty vec.
- Query longer than text â†’ no matches â†’ one non-highlighted chunk.
- All-matched text â†’ one highlighted chunk.
- Query at end of text â†’ highlighted tail + no trailing non-highlighted chunk.
- Multi-query overlapping ranges merged into one chunk.
- Fuzzy with `query.chars().count() > text.chars().count()` â†’ no matches
  â†’ one non-highlighted chunk.
- `ignore_case = false` â†’ byte-exact matching, no normalisation pass.
