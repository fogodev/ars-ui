# Task #286 — Implement Table agnostic core

## Context

Task **#286** ("Implement Table agnostic core", 8 pts, complex tier) is the
framework-agnostic state machine for the `Table` data-display component. The
Collections Epic (#53) it depends on closed in 2026-04-08, so it is unblocked.

Spec: `spec/components/data-display/table.md` (1419 lines).

The Table is a stateful machine that owns row/column selection, sort
descriptors, expansion, grid focus, optional column-resize state, and
optional virtual-scrolling metadata. The agnostic core MUST NOT touch the
DOM — adapters apply focus, measure layout, and observe scroll. The core
produces ARIA + `data-ars-*` attributes via `ConnectApi` and the typed
`Api<'_>` so both Leptos and Dioxus adapters can render identical markup.

### Spec drift to fix in this task

The spec is internally inconsistent in several places. Per the CLAUDE.md
rule "Resolve the spec/code mismatch in the same task," every drift below
is fixed in this PR — the spec edits land alongside the code:

1. **`Context::dir: Direction`** missing from §1.3 but referenced by
   `Api::on_cell_keydown` for RTL arrow-key resolution. Add the field and
   resolve `Direction::Auto → concrete` in `init` from `env.locale`. Add
   `Event::SetDirection(Direction)` so runtime changes are wired through.
2. **`Context::is_sorting: bool`** described in §1.1.1 but absent from
   §1.3. Add it; toggle inside the `SortColumn` arm.
3. **§1.1.1 mentions `Scrolling` state** but §1.1 lists only `Idle`.
   Resolve by deleting the misleading reference in the spec (the
   single-`Idle` design is correct — sort lives in Context, scroll lives
   in adapter).
4. **§1.7 `Event::FocusCell` writes `Some((c, 0))`** — a literal bug that
   hard-codes row 0. Fix the spec **and** code: `Event::FocusCell` carries
   `{ row: Key, col: usize, row_index: usize }`, and the transition writes
   `(col, row_index)` to `focused_cell`.
5. **§1.2 events table omits `FocusRow` and `FocusCell`** even though
   §1.7 defines them. Add both rows to §1.2.
6. **§3.4.3 `build_sort_announcement`** references undeclared `locale` and
   uses `SortDescriptor::column_key` (the real field is `column`). Rewrite
   that code block in the spec to compile against the real `SortDescriptor`
   API. (Adapter helper, not in the agnostic core — but it must compile
   under `cargo xtask spec compile-snippets`.)

## Approach

Implement the Table machine + connect API as a single Rust module living
in a `table/` directory next to existing data-display components, with
unit + snapshot tests inline via `mod tests;`. Use **Tabs** as the
structural template (`crates/ars-components/src/navigation/tabs/`) — it is
the closest existing complex component (registered keys + selection +
keyboard nav + Bindable controlled/uncontrolled + Direction).

### Row registry

The machine needs to know the full row list to validate
`disallow_empty_selection`, derive `aria-rowcount` semantics, and prune
`default_selected_rows` against `disabled_keys` at init. Mirror Tabs'
`Event::SetTabs(Vec<TabRegistration>)`:

- Add **`Event::SetRows(Vec<Key>)`** — replaces `ctx.rows` and snaps
  selection / expansion to the new set.
- Add `Context::rows: Vec<Key>`. The keyboard helpers
  (`on_row_keydown`, `on_cell_keydown`) keep their `all_row_ids: &[&Key]`
  parameter (adapter slice for that render) — the registry exists for
  invariants, not for navigation walks.
- Init prunes any key that appears in both `disabled_keys` and
  `default_selected_rows` from the seed selection (workspace convention;
  spec is silent — document the choice inline).

### Prop sync (controlled-mode parent updates)

Tabs has `Event::SyncProps` plus `Event::SyncControlledValue`. Table has
three Bindable fields, so the plan uses one coalesced event per Bindable
plus a general `SyncProps` for non-Bindable runtime changes:

- **`Event::SyncProps`** — re-applies `selection_mode`,
  `selection_behavior`, `disabled_keys`, `disallow_empty_selection`,
  `escape_key_behavior`, `interactive`, `sticky_header`,
  `min_column_width`, `column_resize_step` from props. Re-runs the
  selection invariant.
- **`Event::SyncControlledSelectedRows(Option<selection::Set>)`**
- **`Event::SyncControlledExpandedRows(Option<BTreeSet<Key>>)`**
- **`Event::SyncControlledSortDescriptor(Option<SortDescriptor<String>>)`**

Each `SyncControlled*` calls `Bindable::sync_controlled` on the matching
Context field.

### Escape-key handling

§1.5 names `EscapeKeyBehavior::{ClearSelection, None}`. The transition
table in §1.7 omits it — add:

- **`Event::EscapeKey`** — when `escape_key_behavior == ClearSelection`
  and the current selection is non-empty, dispatch the same context
  mutation as `DeselectAll` (subject to `disallow_empty_selection`).
  Otherwise no transition.

### Module layout (`table/mod.rs`)

Single file, ~1500–2000 LOC, organized in the same section order as
`tabs/mod.rs`:

1. `State` — single `Idle` variant per §1.1.1 (sort state is in Context).
2. `Event` — every event from §1.2 plus `SetDirection`, `ColumnResize`,
   `SetRowCounts`, `SetLoading`.
3. `EscapeKeyBehavior` — enum (ClearSelection / None) from §1.5.
4. `Messages` — five `MessageFn` fields (select_all, select_row,
   sort_ascending, sort_descending, sort_none); `ComponentMessages`
   marker; `Default` returns static-str fns.
5. `Props` — every field from §1.5 + `disallow_empty_selection`,
   `loading`, `virtual_scrolling`, `total_rows`, `total_cols`,
   `dir: Direction`. `#[derive(ars_core::HasId)]`.
6. `Context` — every field from §1.3 plus the §6/§3.5/§1.6/spec-drift
   fields described above. Derives `Clone`, `Debug`, `PartialEq` (not
   `Eq` — `MessageFn` only ptr-compares).
7. `Part` — `#[derive(ComponentPart)] #[scope = "table"]` with all 14
   anatomy parts + `ColumnResizeHandle { column: String }`.
8. `Machine` — empty marker struct; impl `ars_core::Machine`. Uses
   `ars_core::NoEffect` for `type Effect` (no DOM effects — adapters
   read `ctx.focused_cell`/`ctx.focused_row`/`ctx.focused_col` and apply
   focus directly).
9. `Api<'a>` — every method from spec §1.8 plus the §6 resize-handle
   attrs and §3.5 virtual-scrolling additions to `table_attrs` and
   `row_attrs(row_id, index)`.

### Transitions (high-level)

Match `tabs::Machine::transition` style — each arm returns
`Some(TransitionPlan::context_only(move |ctx| ...))` or `None` when
guarded out:

| Event | Behavior |
|-------|----------|
| `SortColumn { column }` | Toggle: None → Ascending → Descending → None. Update `ctx.sort_descriptor`, set `ctx.is_sorting = true`. |
| `SelectRow / DeselectRow / ToggleRow` | Reject when `selection_mode == None` or row in `disabled_keys`. Honor `disallow_empty_selection` on deselect: if removing the last selected row, return `None`. Delegate to `selection::State::{select, deselect, toggle}` and write `selected_keys` back to `Bindable`. |
| `SelectAll` | Reject unless `Mode::Multiple`. Call `selection::State::select_all()`. |
| `DeselectAll` | Reject when `Mode::None`. Reject when `disallow_empty_selection` and selection is non-empty (return `None` to preserve last). |
| `ExpandRow / CollapseRow` | Skip when no-op (already in/out of set). Mutate cloned `BTreeSet<Key>`. |
| `Focus { cell }` / `FocusRow` / `FocusCell` | Reject when `!ctx.interactive`. Update `focused_cell`/`focused_row`/`focused_col`. |
| `Blur` | Always: clear all three focus fields. |
| `RowAction(key)` | Reject when disabled. No mutation — adapter listens. |
| `SetDirection(dir)` | Idempotent. Replace `ctx.dir` only if new value differs. |
| `SetRows(rows)` | Replace `ctx.rows`. Prune `selected_rows` / `expanded_rows` of keys not present. Clear `focused_row`/`focused_cell` if pointing at a removed row. |
| `EscapeKey` | When `escape_key_behavior == ClearSelection` and selection is non-empty, clear (subject to `disallow_empty_selection`). |
| `ColumnResize { column, width }` | Clamp to `props.min_column_width` (default 50). Upsert `ctx.column_widths[column]` (create-on-first-resize) and set `ctx.resizing_column = Some(column)`. |
| `ColumnResizeEnd { column }` | Clear `ctx.resizing_column` when it matches `column`. |
| `SetRowCounts { total_rows, total_cols }` | Replace `ctx.total_rows`/`ctx.total_cols`. |
| `SetLoading(bool)` | `ctx.loading.set(value)`. |
| `SyncProps` | Re-apply non-Bindable Props fields; re-prune selection/expansion against new `disabled_keys`. |
| `SyncControlledSelectedRows(opt)` | `ctx.selected_rows.sync_controlled(opt)`. |
| `SyncControlledExpandedRows(opt)` | `ctx.expanded_rows.sync_controlled(opt)`. |
| `SyncControlledSortDescriptor(opt)` | `ctx.sort_descriptor.sync_controlled(opt)`. |

### Api / attribute methods

Every method from §1.8 verbatim, plus:

- `column_resize_handle_attrs(column: &str, current_width: f64) -> AttrMap`
  — emits the standard `data-ars-scope="table"` / `data-ars-part=
  "column-resize-handle"`, plus `role="separator"`,
  `aria-orientation="vertical"`, `aria-valuenow=width`, and
  `data-ars-resizing` when `ctx.resizing_column == Some(column)`.
- `on_resize_handle_keydown(column: &str, current_width: f64,
  data: &KeyboardEventData)` — emits `Event::ColumnResize` with
  `width ± ctx.column_resize_step` on `ArrowLeft`/`ArrowRight`,
  RTL-aware via `ctx.dir`.
- `table_attrs` — adds `aria-rowcount` / `aria-colcount` when
  `ctx.virtual_scrolling`.
- A virtual-scrolling-aware `row_attrs(row_id, row_index: usize)` that
  emits `aria-rowindex` (1-based) when `ctx.virtual_scrolling`. The
  non-virtualized `row_attrs(row_id)` from §1.8 is kept as a thin
  delegate that passes `row_index = 0`.
- `select_all_attrs` short-circuits to an empty `AttrMap` when
  `props.select_all_mode == SelectAllMode::None`, and substitutes the
  `AllData { total_count }` count into the `aria-label` via
  `Messages::select_all`.
- `column_header_attrs(column, sortable=false)` MUST omit `aria-sort`
  entirely (§3.4.2). The spec's example sets `aria-sort="none"`
  unconditionally — that is wrong; the named test
  `table_column_header_non_sortable_omits_aria_sort` locks the correct
  behavior.
- `on_cell_keydown` resolves left/right against `ctx.dir == Direction::Rtl`,
  handles `Ctrl+Home`/`Ctrl+End` per §3.2 in addition to the §1.8
  Home/End/Arrow set.
