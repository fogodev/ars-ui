# Task #289 — Implement DatePicker agnostic core

## Context

GitHub issue #289 (epic #224, 8 pts) asks for the framework-agnostic `DatePicker`
state machine: a composite that pairs a date **text input** with an embedded
`Calendar` in a popover, owning popover open/close, date selection, input
parsing/validation, value↔calendar sync, and ARIA/data attributes. Dependencies
#262 (DateField) and #280 (Calendar) are merged. Adapters (Leptos/Dioxus) are
explicitly out of scope (separate tasks); this is **unit-tested agnostic core only**.

Spec: `spec/components/date-time/date-picker.md`. Shared types: `spec/shared/date-time-types.md`.

The spec's `§1.7 Full Machine Implementation` was written before the core engine
and sibling components solidified, so it contains several **compile-blockers and
omissions** that must be fixed in code and synced back into the spec in this same
PR (CLAUDE.md: "resolve spec/code mismatch in the same task"; the issue's
"Spec impact: No spec change required" is the author's expectation, not a
prohibition — the spec literally will not compile as written).

## Decisions locked with the user

1. **Full controlled-prop sync** — add a `SyncProps(Box<Props>)` event + implement
   `on_props_changed` + `initial_effects`, matching Calendar/DateField/Popover.
2. **`aria-haspopup="dialog"` on BOTH Input and Trigger** — keep it on Input (spec)
   and add it to `trigger_attrs()` (issue's named test + Popover precedent); update
   spec §1.8 / §3.1.
3. **Format-pattern-aware** `format_date`/`parse_date` — derive field order +
   separator from the `format` string (MDY / DMY / YMD), so the computed `format`
   field and locale defaults are not dead code. Dependency-free; passes the en-US
   `MM/dd/yyyy` test.

## Spec drifts to fix (code + spec, same PR)

Verified against the real sources (not training data):

| # | Spec §1.7/§1.8 as written | Reality / fix |
|---|---|---|
| D1 | No `type Effect`; effects use string names + `use_platform_effects()`/`focus_element_by_id` closures in core | `Machine` trait requires `type Effect: EffectName`. Define a typed `Effect` enum and emit `PendingEffect::named(...)`; focus is **adapter-driven** (matches issue element/ref note + `popover`/`dialog` which use `PendingEffect::named(Effect::FocusInitial/RestoreFocus)` exclusively, never platform calls in core) |
| D2 | `is_date_unavailable: Option<fn(&CalendarDate) -> bool>` | Calendar's prop is `Option<calendar::IsDateUnavailableFn>` (= `Callback<dyn Fn(&CalendarDate)->bool + Send+Sync>`). Reuse that type so `calendar_props()` forwards cleanly and `Props` stays `Clone+PartialEq+Debug` |
| D3 | `CalendarDate` field access `d.month.get()`, `d.day.get()`, `d.year`; `NonZero::new(..)`; `nzu8(..)` helper | Fields are `pub(crate)`. Accessors are `year()->i32`, `month()->u8`, `day()->u8` (plain ints). `nzu8` does not exist anywhere. `new_gregorian(i32,u8,u8) -> Result<…>` |
| D4 | `parse_date` builds `Some(CalendarDate::new_gregorian(..))` with `NonZero`+`.expect` | `new_gregorian` returns `Result`; use `CalendarDate::new_gregorian(y,m,d).ok()` |
| D5 | min/max check via `date >= *m`, `date <= *m` | `CalendarDate` has **no** `PartialOrd`/`Ord`. Use `date.compare(m) -> Ordering` (`!= Less` / `!= Greater`) |
| D6 | `self.ctx.ids.base_id` in `calendar_props()` | No public `base_id` field; accessor is `ids.id()` |
| D7 | `Context` derives `#[derive(Clone, Debug, PartialEq)]` | `Messages` (holds `MessageFn`) is not `PartialEq`; trait only needs `Clone+Debug`. Mirror Calendar: `#[derive(Clone)]` + manual `Debug` (skip messages), drop `PartialEq` |
| D8 | Machine impl omits `on_props_changed`, `initial_effects`, and a `SyncProps` event | Add all three (decision #1) |
| D9 (tests) | `Service::new(props, Env::default(), Default::default())`; bare `svc.send(..)` | `Service::new(props, &Env, &Messages)` by ref; `send` is `#[must_use]` → `drop(svc.send(..))`; use a `date(y,m,d)` helper like Calendar tests |

## Design (agnostic core)

Mirror the structure of `crates/ars-components/src/date_time/calendar/mod.rs`.

- **State** `enum { Closed, Open }` (derive `Clone, Debug, PartialEq, Eq`).
- **Event** `{ Open, Close, Toggle, SelectDate{date:CalendarDate}, InputChange{value:String},
  FocusIn, FocusOut, KeyDown{key:KeyboardKey}, SyncProps(Box<Props>) }` (+`SyncProps`, D8).
- **Effect** (new, D1) typed enum, `#[derive(Clone, Copy, Debug, PartialEq, Eq, Hash)]`:
  `FocusCalendar` (move focus into the embedded calendar grid/content),
  `RestoreFocusToTrigger` (Close / Escape), `RestoreFocusToInput` (SelectDate when
  `close_on_select`). Adapter implements the actual DOM focus from the named intent.
- **Context** — as spec §1.3 (value `Bindable<Option<CalendarDate>>`, open `Bindable<bool>`,
  input_text, parsed_date, locale, messages, format, min, max, disabled, readonly,
  open_on_click, is_touched, name, required, is_rtl, ids) minus `unavailable_dates`
  (forwarded predicate makes the cache dead — confirm/drop in spec). Derive per D7.
  Keep helper methods `formatted_value`, `parse_input`, `sync_input_text`, and the
  free `format_date`/`parse_date` (now format-aware, decision #3, D3–D5).
- **Props** — as spec §1.4, but `is_date_unavailable: Option<calendar::IsDateUnavailableFn>`
  (D2). Public fields + `Default` (no builder — matches spec; builder is an adapter-era nicety).
- **Machine**:
  - `init`: build `value`/`open` Bindables, compute locale-default `format`, seed
    `input_text`, `initial_state = if *open.get() { Open } else { Closed }`.
  - `transition`: per spec §1.7 with D1/D3/D4/D5 fixes — `Open`/`FocusIn(open)` →
    `.with_effect(PendingEffect::named(Effect::FocusCalendar))`; `Close` →
    `RestoreFocusToTrigger`; `SelectDate` close → `RestoreFocusToInput`; `FocusOut`
    close → no focus effect; `KeyDown` Escape/ArrowDown delegate; `disabled` guard
    returns `None`; `readonly` blocks open/select/input. Add `SyncProps` arm: re-derive
    controlled/uncontrolled Bindables + cached fields from new props (preserving
    uncontrolled internal state, like Calendar), recompute target state from controlled
    `open`, attach Focus/Restore effect only when open actually transitions.
  - `on_props_changed` (D8): `debug_assert!` id stable; return
    `vec![Event::SyncProps(Box::new(new.clone()))]` when `old != new`.
  - `initial_effects` (D8): when `state == Open`, return
    `vec![PendingEffect::named(Effect::FocusCalendar)]` (Popover precedent).
  - `connect`: `Api { state, ctx, props, send }`.
- **Part** (`#[derive(ComponentPart)] #[scope="date-picker"]`): `Root, Label, Control,
  Input, Trigger, ClearTrigger, Positioner, Content, Description, ErrorMessage, HiddenInput`.
- **Api** — all attr getters per spec §1.8 with fixes: `input_attrs` keeps
  `aria-haspopup="dialog"`; `trigger_attrs` **adds** `aria-haspopup="dialog"` (decision #2);
  `hidden_input_attrs` uses `d.year()/d.month()/d.day()` ISO (D3); `calendar_props()` uses
  `ids.id()` (D6) + `IsDateUnavailableFn` (D2) + `..calendar::Props::default()`. Handlers
  (`on_trigger_click`, `on_input_change`, `on_clear_trigger_click`, `on_input_keydown`,
  `on_focusin`, `on_focusout`, `on_content_keydown`, `on_trigger_keydown`) and accessors
  (`is_open`, `selected_date`, `formatted_value`, `open/close/toggle`) per spec.
  `impl ConnectApi { type Part = Part; fn part_attrs(..) }` (matches the 2-method trait).
- **Messages** — `trigger_label`, `clear_label`, `content_label`, `selected_date_label`
  with `MessageFn::static_str`/`MessageFn::new`; `impl ComponentMessages`.

## Out of scope (spec-authoritative)

- **Presets** (issue "tests to add first" mentions Today/Yesterday): spec §6.2/§6.4
  explicitly excludes `PresetTrigger` ("Gaps: None worth adopting"). Not implemented; no test.
- **Live focus / positioning / outside-dismiss / scroll**: adapter concerns (issue
  element/ref note). Core only emits named `Effect`s + stable IDs.
