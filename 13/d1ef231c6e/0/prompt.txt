# Tabs Public Parts And Styled Component Split Plan

## Summary
- Remove the closed-anatomy `Tabs` component from the adapter crates as a public API.
- Make `ars-leptos` and `ars-dioxus` expose only unstyled Tabs primitives and typed collection/rendering contracts.
- Add the closed, ready-made high-level Tabs component under `ars-leptos-components` and `ars-dioxus-components`, alongside CSS and Tailwind source-template variants, composing the new lower-level adapter parts.

## Adapter Crate API
- Replace public adapter `Tabs` with public unstyled parts:
  - `Root<K>`, `List<K>`, `Panels<K>`, `TabShell<K>`, `Tab<K>` part, `CloseTrigger<K>`, `Panel<K>`, `Indicator`, and `LiveRegion`.
  - `Root<K>` receives `tabs: TabsSource<K>` and owns machine setup, registration, callbacks, i18n messages, selection, close, reorder, panel presence, indicator state, refs, and typed context.
  - `List` and `Panels` iterate the root `TabsSource<K>`; consumers do not manually mirror the tab list.
  - `TabShell`, `Tab`, `CloseTrigger`, and `Panel` take a typed `TabRenderItem<K>` supplied by `List` / `Panels`.
- Preserve row-data compatibility where possible:
  - Internally rename row data to `TabItem<K>`.
  - Keep `pub type TabData<K> = TabItem<K>` and, if Rust/framework naming permits, keep `pub type Tab<K> = TabItem<K>` for old row construction.
  - The public trigger part should be named `Tab` if it does not break source compatibility; otherwise use `Trigger` and document the naming decision in specs. Do not use string keys to avoid the naming issue.
- Renderer customization:
  - Dioxus `List` / `Panels` expose renderer props shaped as `Callback<TabRenderItem<K>, Element>`.
  - Leptos `List` / `Panels` expose equivalent cloneable renderer props returning view content, using the repo’s existing `ViewFn`/callback wrapper pattern.
  - Default `List` renders `TabShell { Tab, optional CloseTrigger }` plus `Indicator`; default `Panels` renders `Panel` for every item.
- Remove public closed-anatomy adapter `Tabs` from prelude and docs. If a short compatibility transition is unavoidable, keep it deprecated and hidden from prelude, but the target contract is no public adapter `Tabs`.

## Styled Components Crate
- Add category-first modules:
  - `crates/ars-leptos-components/src/navigation/tabs/{css.rs,tailwind.rs,tabs.css,mod.rs}`
  - `crates/ars-dioxus-components/src/navigation/tabs/{css.rs,tailwind.rs,tabs.css,mod.rs}`
- Implement closed high-level styled components there:
  - `css::Tabs` and `tailwind::Tabs` receive the ergonomic closed-anatomy props previously offered by the adapter wrapper.
  - They compose adapter `Root`, `List`, `Panels`, `TabShell`, `Tab`/`Trigger`, `CloseTrigger`, `Panel`, `Indicator`, and `LiveRegion`.
  - CSS variants include documented part/state selectors; Tailwind variants keep class strings inline in markup per workflow rules.
- Update widgets:
  - Public visual widget demos use styled `ars-*-components` Tabs.
  - Add at least one low-level unstyled composition demo or fixture proving direct adapter parts are usable, but avoid rebuilding behavior locally.

## Spec And Sketch Updates
- Update agnostic and adapter specs:
  - remove monolithic adapter `Tabs` API snippets;
  - document the part API, typed `TabRenderItem<K>`, collection-driven renderers, and no manual key re-listing;
  - define `TabsSource<K>` as the only tab collection/order source;
  - describe fallback rendering for `List`/`Panels` and renderer override behavior;
  - document any private/non-public structural exceptions with reasons.
- Add/update styled-component specs for Leptos/Dioxus components:
  - closed `css::Tabs` / `tailwind::Tabs` live only in `ars-*-components`;
  - styled modules are source templates, not adapter primitives.
- Update `docs/implementation/sketches/tabs-counterpart-audit-sketch.md`:
  - mark old monolithic-adapter conclusion as superseded;
  - record public parts as the new adapter outcome;
  - record styled closed Tabs as the high-level widget/source-template outcome.

## Test Plan

  - SSR tests prove each public part renders correct roles, ARIA, ids, `data-ars-part`, typed item association, consumer styling attrs, and fallback/default renderer output.
  - SSR tests prove `List`/`Panels` iterate `TabsSource<K>` and do not require duplicated key enumeration.
  - Wasm tests prove selection, keyboard roving, manual activation, close click, Delete/Backspace close, dynamic push/pop/reorder, lazy/unmount panels, drag image cloning, and Ctrl+Arrow reorder through public parts.
  - Prelude tests prove only unstyled primitives and row-data types are exported from adapter preludes.
- Styled component tests:
