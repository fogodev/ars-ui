# Plan: Finish ArsProvider Leptos (#347) + Dioxus (#477) adapters

## Context

The two adapter tasks ship the **ArsProvider** component shell — the single root context provider for every ars-ui app (locale, direction, color mode, disabled/read-only, ID prefix, portal/root nodes, platform effects, intl backend, i18n registries, style strategy, plus `dioxus_platform` on Dioxus). The agnostic core (`ArsContext`, `ColorMode`, `PlatformEffects` in `ars-core`) landed via #213; the reactive shell, hooks, and context publishing landed via #190 (Leptos) and #193 (Dioxus). #477 was then closed as superseded by #193, but the discrete *adapter component* deliverable for both frameworks was never finished:

- No dedicated `tests/ars_provider*.rs` integration tests in either adapter crate.
- `ArsProvider` is not exposed in either adapter's `prelude` — every other delivered component module is.
- Spec §28 mandates oracles for SSR rendering, nonce auto-render when `StyleStrategy::Nonce`, default `StyleStrategy::Inline`, and (Dioxus) feature-flag-based `dioxus_platform` defaults — none of these are exercised end-to-end through the public component.

The component implementations themselves (`crates/ars-leptos/src/provider.rs` lines 169-254, `crates/ars-dioxus/src/provider.rs` lines 28-239) and ~40 inline unit/wasm tests already exist; this PR adds the missing integration-tier surface so #347 and #477 can close against a green CI + Codex 👍.

## Scope

One bundled PR closing **#347** and **#477** (reopen #477 first). Add adapter-tier integration test files, expose `ArsProvider` in both preludes, fill the spec §28 gaps. No new dependencies. No spec changes (specs already authoritative). No changes to the component implementations or inline tests.

## Files to modify

### Leptos (#347)

1. **Create** `crates/ars-leptos/tests/ars_provider.rs` — native SSR oracle, gated `#![cfg(all(not(target_arch = "wasm32"), feature = "ssr"))]`, mirroring the `client_only.rs` shape. Scenarios:
   - Renders `<div dir="ltr">` wrapper with children for default props (no locale supplied).
   - Renders `<div dir="rtl">` when an RTL locale (e.g. `Locale::parse("ar-EG")`) is provided.
   - Renders `<div dir="rtl">` when explicit `direction=Signal::stored(Direction::Rtl)` overrides an LTR locale.
   - When `style_strategy=StyleStrategy::Nonce("test-nonce")`, the SSR HTML contains a `<style nonce="test-nonce">` tag (sourced from `ArsNonceStyle`).
   - When `style_strategy=StyleStrategy::Inline` (default), no `<style nonce="…">` is emitted.

2. **Create** `crates/ars-leptos/tests/ars_provider_wasm.rs` — `#![cfg(target_arch = "wasm32")]` + `wasm_bindgen_test_configure!(run_in_browser)`. Use a locally-defined `#[component] fn Probe()` that reads `use_context::<ArsContext>()` (reuse the pattern from the existing inline `ConfiguredProviderProbe`). Scenarios that complement (don't duplicate) the inline tests:
   - All-props configuration round-trip: `locale`, `direction` override, `color_mode`, `disabled`, `read_only`, `id_prefix`, `portal_container_id`, `root_node_id`, `style_strategy=Cssom` — every field observable via the probe matches the input.
   - Nonce auto-render: when `style_strategy=StyleStrategy::Nonce(...)` is passed, a probe verifies (a) `ArsNonceCssCtx` is published and (b) the rendered DOM contains a `<style>` element with the expected nonce attribute (mount via `leptos::mount::mount_to(...)` then query).
   - `use_style_strategy()` fallback: outside any provider, returns `StyleStrategy::Inline`.

3. **Edit** `crates/ars-leptos/src/prelude.rs` — add `pub use crate::ArsProvider;` and `pub use ars_core::ColorMode;` (color mode is a configuration type users pass via the `color_mode` prop; add a short doc comment on the `ColorMode` line to explain why it's in the prelude).
