# Implement Tasks #233, #235, #236 — Four Agnostic Input Components

## Context

Three GitHub issues in Epic #220 (Agnostic input components) ship four framework-agnostic state machines into `crates/ars-components/src/input/`:

- **#233** — PasswordInput + SearchInput (5 pts) — adds masked/visible toggle and clear/search behavior on top of the text-field pattern.
- **#235** — NumberInput (5 pts) — spinbutton with min/max/step, scrubbing, locale-aware parse/format.
- **#236** — PinInput (5 pts) — multi-cell OTP/PIN with roving tabindex, auto-advance, paste distribution, hidden-input form submission.

These are the next ready, unblocked tasks in the roadmap: all 5pts, all `Layer: Component`, all `Framework: None`, all `Depends on: None`. The closest precedent for the implementation pattern is `crates/ars-components/src/input/text_field/mod.rs` (1866 LOC) for the public surface, `crates/ars-components/src/utility/toggle.rs` for the 2-state machine, and `crates/ars-components/src/overlay/tooltip.rs` for the `PendingEffect`-driven debounce timer that SearchInput needs.

Goal: land all four components in a single bundled PR (precedent: PR #654 bundled Toggle + Swap), closing #233/#235/#236.

## Resolved Design Decisions

1. **PR strategy**: one bundled PR. Title `[codex] Add PasswordInput, SearchInput, NumberInput, and PinInput agnostic cores`; body contains `Closes #233`, `Closes #235`, `Closes #236`.
2. **`#[derive(ComponentPart)]` supports struct variants** — proven by `crates/ars-derive/src/component_part.rs:531-553` (test derives `enum Part<T> { Root, Trigger(T), Panel { index: usize } }`). PinInput's `Input { cell_index: usize }` works as-is.
3. **NumberInput's `FormatOptions` unifies on the existing `ars_i18n::number::FormatOptions`** rather than defining a duplicate. The spec currently inlines its own definition with overlapping but differently-named fields — per CLAUDE.md "If code and spec disagree, resolve the mismatch in the same task or PR", I'll update `spec/components/input/number-input.md` §1.3 to re-reference `ars_i18n::number::FormatOptions`. Single source of truth, no duplication.
4. **Round-half-up**: `ars_i18n::number::RoundingMode::HalfUp` (crates/ars-i18n/src/number/mod.rs:127). Configured on `FormatOptions.rounding_mode`.
5. **Locale-aware parse**: `ars_i18n::number::parse_locale_number(input, locale) -> Option<f64>` (line 569) for NumberInput's `Change` handler.
6. **SearchInput debounce**: mirror `tooltip::Effect::{OpenDelay, CloseDelay}` pattern — emit `Effect::SearchDebounce` via `.with_effect(PendingEffect::named(Effect::SearchDebounce))`, cancel via `.cancel_effect(Effect::SearchDebounce)`. Adapters own the wall-clock timer and send `Event::DebounceExpired` back.
7. **PinInput callbacks in Props**: TextField proves `Callback<dyn Fn(...) + Send + Sync>` derives `Clone + Debug + PartialEq` cleanly — use `#[derive(Clone, Debug, PartialEq, HasId)]` first; only fall back to manual impls if rustc rejects it.
